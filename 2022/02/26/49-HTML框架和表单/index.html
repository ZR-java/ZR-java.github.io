    一、HTML的框架(已弃用)

    框架(分帧)的作用

    • 将一个浏览器文档窗口分隔成多个窗口，每个窗口中都可 以显示一个独立的网页文件

    框架中的各个页面

    index.html

    <frameset rows="20%,*,10%">
    <frame src="top.html">
    <frameset cols="20%,*">
        <frame src="left.html">
        <frame name="content" src="main_emp.html">
    </frameset>
    <frame src="bottom.html">
</frameset>

    left.html

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>框架学习</title>
</head>
<body>
    <h3>功能菜单</h3>
    <a href="main_dept.html" target="content">部门管理</a><br/>
    <a href="main_position.html" target="content">职位管理</a><br/>
    <a href="main_salary.html" target="content">薪资管理</a><br/>
    <a href="main_emp.html" target="content">员工管理</a><br/>
    <a href="main_checkingIn.html" target="content">考勤管理</a><br/>
    <a href="main_advice.html" target="content">意见管理</a><br/>
    <a href="main_permission.html" target="content">权限管理</a><br/>
</body>
</html>

    <frameset></frameset>标签

    • 框架标签，框架集的根标签，可以指定子页面、框架集的排列方式和分布的比例
    • 可以是html标签的子标签，也可以作为根标签
    • 不可以作为body标签的子标签
    • 除了frame标签与frameset标签之外不可以再包含其他的html标签

    属性

    • cols
      • 垂直方向上分配子窗口，像素或者是父窗口的百分比
      • 使用“像素数”和%分割左右窗口，“*”表示剩余部分，比如：如果使用”*,*“表示框架平均分成2个部分，如果使用“*,*,*“表示平均分成三个部分
    • rows
      • 水平方向上分配子窗口，…….同上
    • frameborder
      • 指定是否显示边框，默认值为1，0-不显示
    • border
      • 设置边框的大小，默认值5像素

    <frame/>标签

    • 具体的一个帧，由该标签绑定具体的网页， <frame>标记是一个单标记，该标记必须放在<frameset></frameset>中使用， 在<frameset>中设置了几个窗口，就必须对应使用几个<frame>框架，而且还必须使用src属性指定一个网页

    属性

    • src
      • 网页URL地址
    • name
      • 名称，target指向该名称的网页会替换
    • scrolling
      • 是否显示子窗口的滚动条，默认是自适应，auto-根据需求自动出现，yes-有，no-无
    • noresize
      • 是否允许使用鼠标拖拽子窗口改变其大小，不设置该属性表示可拖拽，设置该属性（值一定是noresize）表示不可拖拽
    • frameborder
      • 指定是否显示边框，1-显示，0-不显示

    <noframes></noframes>标签

    • 如果客户端的浏览器不支持框架集，可以使用该标签，在里面注明一些提示性的文字；如果支持，这些文字则不会 显示

    <iframe></iframe>标签

    • 不属于frameset标签的子标签
    • 在一个已经存在的网页中使用<iframe>再嵌套一个页面，实现画中画的感觉

    属性

    • 类似<frame>标签

    二、HTML的表单

    <form></form>标签

    • 被form标签所包裹的表单标签，用户填写好信息之后， 点击链接或者按钮，会将这些信息发送到服务器，服务器端程序根据这些信息处理完业务逻辑之后，再返回客户端一个新的页面
    • 属性
      • action.
        • 设置接收或者处理浏览器递交的表单内容的服务器程序的URL，如果为空则使用当前文档的URL地址，如果表单中不需要使用action属性 也要指定其属性为”no”
      • method
        • 定义浏览器将表单中的信息提交到服务器端的处理程序的方式，一般取值为post或get。.
        • get方式提交时，会将表单的内容附加在URL地 址的后面，所以限制了提交的内容的长度，不超过8192个字符，在url的后面能看见，所以不具备保密性。.
        • post方式提交时，将表单中的数据一并包含在表单的头中，一起传送到服务器中处理，没有数据大小限制，因为在html的头中，所以看不见，具有保密性.
      • target
        • 指定服务器返回结果显示的目标窗口或者框架集窗口
      • title
        • 设置当网站访问者的鼠标在表单上的任一位置停留几秒钟时，浏览器用黄色小浮标显示的文本
      • enctype
        • 浏览器使用哪种编码方式将表单数据发送到服务器
          • application/x-www-form-urlencoded（默认值）
          • multipart/form-data（上载时需要改成该值）

    <input></input>标签

    • 输入表单标签，根据type的取值不同，显示不同的效果
    • 属性
      • name
        • 为该标签取名称，后台程序会根据该名称获取值
      • value
        • 该标签所显示的值
      • size
        • 调整输入框的宽度
      • maxlength
        • 可提供输入文本的最长长度
      • disabled
        • 是否可获取焦点，设置该属性并且值为disabled表示不可获取焦点，不设置则可以获取焦点
      • readonly
        • 是否可编辑，设置该属性并且值为readonly表示只读，不设置则不是 只读
      • checked
        • 单选按钮、复选框是否被选中，设置该属性并且值为checked表示选中，不设置则不选中
      • type
        • 可以有如下一些取值，不同的取值可以显示不同外观
        • text：文本
        • password：密码
        • radio：单选按钮
        • checkbox：复选按钮
        • image：显示一个图像，src属性指定图像的具体位置。该标签也有submit提交按钮的功效，会将所有信息已经鼠标点击图片时的x、y轴的坐标发送到服务器端
        • file：显示一个不可编辑的文本输入框与一个文件选择器的按钮，选择好的文件的绝对路径的地址会显示在文本框中
        • submit：提交按钮
        • reset：重置按钮
        • button：普通按钮
        • hidden：隐藏域

    <select></select>标签

    • 下拉列表，option子标签提供下拉列表项
    • 属性
      • name：取名称
      • multiple：是否提供可以多选，取值为multiple表示可以多选，如果不设置则只能选择一个
      • size：设置列表框中可以看见的列项，默认值为1

    • option子标签

      • option的value属性
        • 选中该选项的值
      • option的selected属性
        • 取值selected为默认选中，如果不设置该属性则不选中

    <textarea></textarea>标签

    • 文本输入域，多行文本域
    • 该标签无value属性，默认值可以在起始标签与结束标签之间填写
    • 属性
      • name：取名称
      • cols：输入域所占的列数
      • rows：输入域所占的行数
    • 禁止拖拽
      • style=”resize:none;”
