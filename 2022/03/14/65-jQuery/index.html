一、jQuery简介

  • JavaScript库

    • 为了简化JavaScript的开发，一些JavsScript的库就诞生了
    • JavaScript库封装了很多预定义的对象和实用函数，能帮助使用者建立有高难度交互的Web2.0特性的富客户端页面，并且兼容各大浏览器

  • 流行的JavaScript库

    • jQuery、EXT_JS、DWR、Prototype、Dojo、 MooTools、YUI

  • jQuery是继Prototype之后又一个优秀的JavaScript库，是一个由John Resig创建于2006年1月的开源项目

  • 现在的jQuery团队主要包括核心库、UI和插件等

  • jQuery凭借着简洁的语法和跨平台的兼容性，极大地简化了JavaScript开发人员查找元素、操作DOM、处理事件、 执行动画和开发Ajax的操作

二、jQuery特点

  • jQuery的宗旨
    • WRITE LESS，DO MORE
    • 写更少的代码，做更多的事情
  • 特点
    • 轻量级
      • jQuery非常轻巧，采用Dean Edwards编写的Packer （http://dean.edwards.name/packer/）压缩后，大小不到 30KB。如果使用Min版并且在服务器端启用Gzip压缩后，大小只有18KB
    • 强大的选择器
      • jQuery允许开发者使用从CSS1到CSS3几乎所有的选择器，以及jQuery独创的高级而复杂的选择器。另外还可以加入插件使其支持XPath选择器，甚至开发者可以编写属于自己的选择器
    • 出色的DOM操作的封装
      • jQuery封装了大量常用的DOM操作，使开发者在编写DOM操作相关程序的时候能够得心应手
    • 可靠的事件处理机制
      • 吸收了javaScript专家Dean Edwards编写的事件处理函数的精华， 使得jQuery在处理事件绑定的时候非常可靠。在预留退路、循序渐进以及非入侵式变成思想方面，jQuery做得非常不错
    • 完善的Ajax
      • 使开发者处理Ajax的时候能够专心处理业务逻辑而无需关心复杂的浏览器兼容性和XMLHttpRequest对象的创建和使用的问题
    • 不污染顶级变量
      • jQuery只建立一个名为jQuery对象，其所有的函数方法都在这个对象之下
    • 出色的浏览器兼容性
      • jQuery修复了一些浏览器之间的差异，使开发者不必在开展项目前建立浏览器兼容库
    • 链式操作方式
      • jQuery中最有特色的莫过于它的链式操作方式 —— 即对发生在同一个jQuery对象上的一组动作，可以直接连写而无需重复获取对象
    • 隐式迭代
    • 行为层与结构层的分离
      • 开发者可以使用jQuery选择器选中元素，然后直接给元素添加事件。这种将行为层与结构层完全分离的思想，可以使jQuery开发人员和HTML或其他页面开发人员各司其职，摆脱开发冲突或个人单干的开发模式
    • 丰富的插件支持，后期扩展非常方便
      • jQuery的易扩展性，吸引了来自全球的开发者编写jQuery的扩展插件。目前已经有超过几百种的官方插件支持，而且还不断有新插件面世
    • 完善的文档
      • jQuery的文档非常丰富

三、DOM和jQuery对象之间的互转

1.DOM对象

  • DOM解析HTML页面，将页面元素解释为元素节点、属性节点和文本节点
  • 而通过DOM解析HTML页面元素，所得到的DOM元素就是DOM对象
  • DOM对象可以使用JavaScript中的方法

2.jQuery对象

  • jQuery对象就是通过jQuery包装DOM对象后产生的对象
  • jQuery对象是jQuery所独有的
  • 虽然jQuery对象是包装DOM对象后产生的，但是jQuery无法使用DOM对象的任何方法，同理DOM对象也不能使用jQuery里的方法，乱使用会报错
  • 规定：如果获取的是jQuery对象，那么要在变量前面加上$

3.DOM对象转成jQuery对象

  • 对于已经是一个DOM对象，只需要用$()把DOM对象包装起来，就可以获得一个jQuery对象

  • 原始javaScript的方式

  • jQuery的方式

4.jQuery对象转成DOM对象

两种转换方式可以将一个jQuery对象转换成DOM对象

  • [index]
    • jQuery对象是一个数组对象，可以通过[index]的方法得到相应的DOM对象
  • .get(index)
    • jQuery本身直接提供get(index)方法得到相应的DOM对象

两种转换方式可以将一个jQuery对象转换成DOM对象

四、jQuery的选择器

什么是jQuery选择器

  • jQuery中的选择器完全集成了CSS的风格
  • 利用jQuery选择器可以非常便捷和快速地找出特定的 DOM元素
  • 学会使用选择器是学习jQuery的基础

jQuery选择器的优势

  • 语法简洁
    • $()函数在很多JavaScript类库中都被作为一个选择器函数来使用，在jQuery中也不例外
  • 支持CSS1到CSS3选择器，jQuery选择器支持CSS1、CSS2的全部和CSS3的部分选择器，同时拥有少量独有的选择器
    • 使用jQuery选择器时无需考虑浏览器是否支持这些选择器
  • 完善的处理机制

1.基本选择器

基本选择器是jQuery中最常用的选择器，也是最简单的选择器，它通过元素id、class、标签名来查找DOM元素（在网页中id只能使用一次，class允许使用多次）

分类

  • id选择器
    • 语法：$(“#div”)
    • 返回值：单个元素的组成的集合
    • 含义：选择网页中id的值为div的元素组成的集合
  • class选择器
    • 语法：$(“.div”)
    • 返回值：集合
    • 含义：选择网页中class的值为div的元素组成的集合
  • Element（元素）选择器
    • 语法：$(“div”)
    • 返回值：集合
    • 含义：选择网页中div的元素组成的集合
  • *选择器
    • 语法：$(“*”)
    • 返回值：集合
    • 含义：选择网页上下文中所匹配的元素组成的集合
  • selector1,selector2 … selectorN选择器（组合选择器）
    • 语法：$(“div,span,p.myClass”)
    • 返回值：集合
    • 含义：将每一个选择器匹配到的元素合并后一起返回，可以指定任意多个选择器，并将匹配到的元素合并到一个结果内

2.层次选择器

如果想通过DOM元素之间的层次关系来获取特定元素, 例如后代元素、子元素、相邻元素、兄弟元素等, 则需要 使用层次选择器

分类

  • ancestor descendant
    • 语法：$(“ancestor descendant”)
    • 返回值：集合
    • 含义：在给定的ancestor元素下所有descendant（后代）元素
  • parent > child
    • 语法：$(“parent > child”)
    • 返回值：集合
    • 含义：在给定的父元素下匹配所有子元素（不包括孙子元素）
  • prev + next
    • 语法：$(“prev+next”)
    • 返回值：集合
    • 含义：匹配所有紧接在prev元素后的next元素（匹配同辈元素）
    • 注意：该选择器只能选择紧贴着prev元素后面的next元素，如果next元素是在prev元素的后面，但是不是紧贴着的，该选择器无效，需要借助方法和过滤选择器解决该问题
  • prev ~ siblings
    • 语法：$(“prev ~ siblings”)
    • 返回值：集合
    • 含义：匹配prev元素后面的所有siblings元素。匹配之后的元素不包含该元素在内，并且siblings匹配的是和prev同辈的元素，其后辈元素不被匹配，该选择器只能选择”#prev”元素后面的同辈元素，而jQuery中的siblings()方法与前后位置无关，只要是同辈节点就可以选取.

3.过滤选择器

过滤选择器主要是通过特定的过滤规则来筛选出所需的DOM元素，该选择器都以”:”开头

按照不同的过滤规则，过滤选择器可以分为如下6种

  • 基础过滤选择器
  • 内容过滤选择器
  • 可见性过滤选择器
  • 属性过滤选择器
  • 子元素过滤选择器
  • 表单对象属性过滤选择器

基础过滤选择器

  • :first
    • 语法：$(“li:first”)
    • 返回值：获取第一个元素
    • 含义：获取匹配的第一个元素
  • ：last
    • 语法：$(“li:last”)
    • 返回值：获取最后个元素
    • 含义：获取匹配的最后个元素
  • :not(selector)
    • 语法：$(“input:not(:checked)”)
    • 返回值：集合
    • 含义：去除所有与给定选择器匹配的元素
  • :even
    • 语法：$(“tr:even”)
    • 返回值：集合
    • 含义：匹配所有索引值为偶数的元素，从0开始计数
  • :odd
    • 语法：$(“tr:odd”)
    • 返回值：集合
    • 含义：匹配所有索引值为奇数的元素，从0开始计数
  • :eq(index)
    • 语法：$(“tr:eq(1)”)
    • 返回值：集合
    • 含义：匹配一个给定索引值的元素，从0开始计数
  • :gt(index)
    • 语法：$(“tr:gt(1)”)
    • 返回值：集合
    • 含义：匹配所有大于给定索引值的元素
  • :lt(index)
    • 语法：$(“tr:lt(2)”)
    • 返回值：集合
    • 含义：匹配所有小于给定索引值的元素
  • :header
    • 语法：$(“:header”)
    • 返回值：集合
    • 含义：匹配如 h1, h2, h3之类的标题元素
  • :animated
    • 语法：$(“:animated”)
    • 返回值：集合
    • 含义：匹配所有正在执行动画效果的元素
  • :focus
    • 语法：$(“:focus”)
    • 返回值：jQuery对象
    • 含义：匹配当前获取焦点的元素

内容过滤选择器

内容过滤选择器的过滤规则主要体现在它所包含的子元素和文本内容上

  • :contains(text)
    • 语法：$(“div:contains(‘John’)”)
    • 返回值：集合
    • 含义：匹配包含给定文本的元素
  • :empty
    • 语法：$(“td:empty”)
    • 返回值：集合
    • 含义：匹配所有不包含子元素或者文本的空元素
  • :has(selector)
    • 语法：$(“div:has(p)”)
    • 返回值：集合
    • 含义：匹配含有选择器所匹配的元素的元素
  • :parent
    • 语法：$(“td:parent”)
    • 返回值：集合
    • 含义：匹配含有子元素或者文本的元素

可见度过滤选择器

可见度过滤选择器是根据元素的可见和不可见状态来选择相应的元素

  • :hidden
    • 语法：$(“tr:hidden”)
    • 返回值：集合
    • 含义：匹配所有不可见元素，或者type为hidden的元素
  • :visible
    • 语法：$(“tr:visible”)
    • 返回值：集合
    • 含义：匹配所有的可见元素

属性过滤选择器

属性过滤选择器的过滤规则是通过元素的属性来获取相应的元素

  • [attribute]
    • 语法：$(“div[id]”)
    • 返回值：集合
    • 含义：匹配包含给定属性的元素
  • [attribute=value]
    • 语法：$(“input[name=’newsletter’]”).attr(“checked”, true);
    • 返回值：集合
    • 含义：匹配给定的属性是某个特定值的元素
  • [attribute!=value]
    • 语法：$(“input[name!=’newsletter’]”).attr(“checked”, true);
    • 返回值：集合
    • 含义：
      • 匹配所有不含有指定的属性，或者属性不等于特定值的元素
      • 此选择器等价于:not([attr=value])
      • 要匹配含有特定属性但不等于特定值的元素，请使用 [attr]:not([attr=value])
  • [attribute^=value]
    • 语法：$(“input[name^=’news’]”)
    • 返回值：集合
    • 含义：匹配给定的属性是以某些值开始的元素
  • [attribute$=value]
    • 语法：$(“input[name$=’letter’]”)
    • 返回值：集合
    • 含义：匹配给定的属性是以某些值结尾的元素
  • [attribute*=value]
    • 语法：$(“input[name*=’man’]”)
    • 返回值：集合
    • 含义：匹配给定的属性是以包含某些值的元素
  • [selector1][selector2][selectorN]
    • 语法：$(“input[id][name$=’man’]”)
    • 返回值：集合
    • 含义：复合属性选择器，需要同时满足多个条件时使用

子元素过滤选择器

  • :nth-child(index)
    • 语法：$(“div :nth-child(2)”)
    • 返回值：集合
    • 含义：
      • 匹配其父元素下的第n个子元素（n从1开始）
      • 这个选择器和之前的基础过滤器中的eq()有些类似，不同的地方就是前者是从0开始，后者是从1开始的
  • :nth-child(even)
    • 语法：$(“div :nth-child(even)”)
    • 返回值：集合
    • 含义：匹配其父元素下的偶数子元素
  • :nth-child(odd)
    • 语法：$(“div :nth-child(odd)”)
    • 返回值：集合
    • 含义：匹配其父元素下的奇数子元素
  • :nth-child(3n)
    • 语法：$(“div :nth-child(3n)”)
    • 返回值：集合
    • 含义：匹配其父元素下的3的倍数的子元素，n从0开始
  • :nth-child(3n+1)
    • 语法：$(“div :nth-child(3n+1)”)
    • 返回值：集合
    • 含义：匹配其父元素下的3的倍数+1的子元素，n从0开始
  • :first-child
    • 语法：$(“div :first-child”)
    • 返回值：集合
    • 含义：匹配第一个子元素
  • :last-child
    • 语法：$(“div :last-child”)
    • 返回值：集合
    • 含义：匹配最后一个子元素
  • :only-child
    • 语法：$(“div :only-child”)
    • 返回值：集合
    • 含义：如果某个元素是父元素中唯一的子元素，那将会被匹配；如果父元素中含有其他元素，那将不会被匹配
  • 注意
    • 子元素过滤器之前有一个空格，不能写成 $(“div:last-child”).

表单对象属性过滤选择器

此选择器主要对所选择的表单元素进行过滤

  • :enabled
    • 语法：$(“input:enabled”)
    • 返回值：集合
    • 含义：匹配所有可用元素
  • :disabled
    • 语法：$(“input:disabled”)
    • 返回值：集合
    • 含义：匹配所有不可用元素
  • :checked
    • 语法：$(“input:checked”)
    • 返回值：集合
    • 含义：匹配所有选中的被选中元素（复选框、单选框等）
  • :selected
    • 语法 ：$(“select option:selected”)
    • 返回值：集合
    • 含义：匹配所有选中的option元素
