    对象声明,变量赋值

    现实世界,先有客观存在,再有抽象

    ​ 对象 类

    java程序中,先有类,再有对象 (编程思想中)

    类:宏观抽象的集合体

    对象:是类的具体表现形式 (客观体表现形式)

    对象:客观世界的存在体,始终存在与计算机的堆内存中

    对象的引用:

    • 是一个变量 存在栈内存中
    • 本身没有任何意义
    • 可以称为引用变量 (对象变量是一个误区)
    • 只有明确指向的引用变量才有真正的意义,否则即为null

    声明对象引用

    引用类型的声明

    • 声明对象引用的方式和基本数据类型的方式很类似
    • Dog dog;

    基本数据类型的声明

    • int age;

    引用变量的赋值

    • 事实上没有对象变量这样的东西存在
    • 只有引用 (reference) 到对象的变量 (一般把这个变量直接叫做引用)
    • 引用并不是对象的容器,而是类似指向对象的指针
    • 引用变量的赋值,就是让该引用变量指向哪一个对象

    创建对象

    • java中的引用变量的目的就是要指向一个java对象

    • java对象的创建,需要使用new关键字

      Dog dog = new Dog();

      • 在堆内存中开辟内存空间,该空间中放置一个Dog对象
      • 在栈内存中定义一个引用变量dog
      • 将dog引用变量指向Dog()对象

    Person.java

    public class Person{
	//成员变量 属性 域(field)
	String name;
	boolean gender;
	int age;
	
    //方法
	void eat(String food){
		System.out.println("人在吃"+food);
	}
	 
	void sleep(){
		System.out.println("人睡觉");
	}
    
	void printInfo(){
	System.out.println("name="+name+",gender="+gender+",age="+age);
	}
    
}

    TestPerson1.java (都是不正规的写法,属性应该private等)

    public class TestPerson1{
	
	public static void main(String[] args){
		
		Person p1 = new Person();
		p1.name = "zhangsan";
		p1.age = 25;
		p1.eat("蔬菜泥");
		p1.printInfo();
		
		//当只需要使用一次的时候使用
		new Person().name = "lisi";
		//new Person().age = 26;	已经new了一个新的对象,这是另一个对象的age属性
	
		Person p2 = null;	//此时的p2没有指向任何的对象,为空指针
		p2.name = "wangwu"; //为什么空引用能够访问对象的属性?编译时不出错,运行时却出错(java.lang.NullPointerException)
	
	}
	
}

    java.lang.NullPointerException : 空指针异常:使用了没有指向任何对象的引用

    对象的生命周期

    生命周期:组件的创建到销毁的整个过程

    GC机制:垃圾回收机制,自行处理内存的回收释放

    TestPerson2.java

    public class TestPerson2{
	
	public static void main(String[] args){
		
		Person p1 = new Person();
		p1.name = "zhangsan";
		
		Person p2 = new Person();
		p2.name = "lisi";
		
		//引用变量之间的赋值,其意义是将被赋值的引用指向同一个对象
		Person p3 = p1;
		p3.age = 25;
		
		p2 = p1;
		/*
		此时第8行的对象已经没有了任何引用,
		将被垃圾回收机制所回收
		*/
		p1.printInfo();
		p3.printInfo();
		
		System.out.println(p1.equals(p3));
		
	}
	
}

    初始化

    当声明完一个变量之后，我们需要为其赋值，这个过程称为变量的初始化

    自动初始化

    仅仅声明，不再为其赋值

    基本数据类型变量的自动初始化有默认值

    • byte short int long 为 0
    • float double 为 0.0
    • char 为 ‘\u0000’
    • boolean 为 false

    引用变量的自动初始化有默认值

    • null

    基本是成员变量

    显式初始化

    改变基本数据类型的默认值

    为引用变量赋值，使其有明确的指向

    如

    • int age = 25;
    • double d = 3.14;
    • boolean gender = true;
    • Student stu = new Student(“zhangsan”,25);
    • Dog dog = new Dog(“旺财”,5);

    基本是局部变量

    null值的引用

    特殊的引用变量值

    明确该引用变量不指向任何的对象

    该引用变量只有编译时类型，没有运行时类型

    编译时类型:就是在编译期间看的类型

    运行时类型:就是在运行期间看的类型
