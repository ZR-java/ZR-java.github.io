13-对象构造和成员变量、局部变量

2021-08-28- 2022-03-10
673- 2m
JavaSE-Java

对象的构造

  • 构造器与类同名
  • 每一个类可以有一个以上的构造器
  • 构造器可以有0个、1个或者1个以上的参数
  • 构造器没有返回值类型，连void也没有
  • 构造器总是伴随着new操作符一同使用，让你有机会介入new的过程

一个对象的创建过程

  • 为对象分配内存空间
  • 调用构造方法
  • 为实例变量赋初始值

把初始化代码放到构造方法中,并且把构造方法设定成需要参数的

TestEmployee.java

import java.util.Date;

public class TestEmployee{
	
	public static void main(String[] args){
		
		Employee e1 = new Employee();
		e1.setName("张无忌");
		e1.setGender(true);
		e1.setBirthday(new Date());
		e1.setWorkNumber("zr-001");
		e1.setSalary(10000);
		
		Employee e2 = new Employee("zhangsan",true,new Date(),"NO-001",10000);
		
		System.out.println("e1="+e1.info());
		System.out.println("e2="+e2.info());
		
	}
	
}
编译器一定会帮你写出没有参数的构造方法吗?不会
  • 如果你已经写了一个有参数的构造方法,并且你需要一个没有参数的构造方法,则你必须自己动手写
  • 如果类有一个以上的构造方法,则参数一定要不一样,我们可以认为这几个构造方法形成重载关系

成员变量、局部变量

成员变量有默认值,局部变量没有

变量按其类型分类

  • 基本类型变量
  • 引用类型变量

变量按其作用域分类

  • 成员变量

    • 类级别的成员变量(被static修饰符修饰)

    • 实例级别的成员变量(没有被static修饰符修饰)

    • 在一个类定义的内部,跟方法属于同一个级别 如:

      class Employee{

      ​ public static int count;

      ​ private String name;

      ​ public String getName(){

      ​ return name;

      ​ }

      }

  • 局部变量

    • 代码块定义的变量
    • 方法内部定义的变量
    • 方法的形参
    • 成员变量、局部变量同名，优先使用局部变量
    • 成员变量的作用域大于局部变量
    • 各个方法、代码块的内部的局部变量没有任何影响

成员变量与局部变量之间的差别

成员变量是声明在类内部而不是方法内

局部变量是声明在方法内部的

局部变量在使用前必需初始化

局部变量没有默认值!如果在变量被初始化前就使用,编译会报错

方法的参数基本上与局部变量相同,但你不需要直接初始化它,编译器会确保方法调用时会有与声明相符的参数传进来,且参数会自动被赋值

局部变量没有访问修饰符
