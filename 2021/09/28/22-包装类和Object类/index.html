包装类

什么是Java包装类?

有时候，需要将int这样的基本类型转换为引用类型，

Java中所有的基本类型都对应了一个对象类，我们称这些类为基本类型的包装类。

Byte、Short、Integer、Long

Float、Double

Character

Boolean

Void

Java类的属性，一般使用包装类型，方法的形参一般使用基本类型。

包装类型与String类型具有相同的“池”的概念。（详见08，equals和==）

String池

常量池

自动封解箱 (JDK1.5之后的新特性)

封箱：把基本类型转换成对应的包装类型。

解箱：把包装类型装换成对应的基本类型。

⭐⭐⭐⭐

public class TestMain{
   public static void main(String[] args){
       String s = "hello";		//字面值
       int i1 = 100;
       //自动封箱
       Integer i2 = 100;	//int ->Integer 	
       //Integer i2 = new Integer(100);
       //自动解箱
       i1 = i2;		//Integer -> int 	
       //int i1 = Integer.parseInt(i2);
       
       //基本类型、包装类型、String类型之间转换
       //基本类型、包装类型，自动进行
       
       //基本类型	-->String
       String s1 = ""+i1;	//推荐这种方法
       String s2 = String.valueOf(i1);
       
       //String -->基本类型
       String s3 = "200";
       int i3 = Integer.parseInt(s3);
       
   } 
}

Object类

Object类是Java所有类的父类，在Java的封装、继承、多态的概念支持下，Object类的一些属性和方法都会起到一些共性的作用。

String toString() : 返回该对象的字符串表示

System.out.println(p1);		//默认调用
System.out.println(p1.toString());	//显示调用(直观地看到)

直接打印对象，相当于打印的是调用该对象的toString方法返回的结果。

int hashCode() : 返回该对象的哈希码值

在java中，尽量让不同的对象有不同的哈希码。

需要重写此方法

boolean equals(Object obj) : 指示其他某个对象是否与此对象”相等“。

需要重写此方法,重写此方法后new一个属性一样的新对象就不会创建两个.

equals()方法是继承的Objec类的，类的属性用的都是包装类，是引用类型，都继承的Object类，都有equals()方法。

比如：String类型的equals()比较的都是字符序列（字符串）

​ Boolean类型的比较的是true或false

​ Integer类型比较的是值

protected void finalize() : 当垃圾回收器确定不存在对该对象的更多引用时,由对象的垃圾回收器调用此方法。

⭐ GC回收机制,不需要我们写,java自己调用

Class getClass() : 返回此Object的运行时类

⭐⭐⭐⭐⭐

请解释：对象、类、类的对象、类对象、类的类

答：三个固定名词（类的类专指Class类，类对象专指蓝色部分，类的对象专指橙色部分。）处于一级抽象时，对象等同于类的对象，类等同于类对象；处于二级抽象时，对象等同于类对象，类等同于类的类。

  1. //根据对象获得类对象的实例，也就是获得类对象。

    Class cls1 = p1.getClass();

    cls1引用变量所指向的对象 是 p1引用变量所指向的对象的类对象是 Class类型的对象。

  2. //根据类名获得类对象的实例

    Class cls2 = Person.class;

  3. //根据类的全名获得类对象的实例

    Class cls3 = Class.forName(“Person”);

后面基本用不到

protected Object clone() : 创建并返回此对象的一个副本

实现克隆的步骤

  • 要克隆类实现Cloneable接口 是一个标记接口(里面没有方法)
  • 覆盖clone()方法
  • 扩展clone()方法的访问修饰符到public

拷贝:

  • 深拷贝：对象中的所有的属性全部都拷贝一份

  • 浅拷贝：对象中的所有的基本数据类型以及常量池（String，Double…)属性拷贝一份，引用类型不拷贝。

    当拷贝一个变量时，原始变量与拷贝变量引用同一个对象。

克隆:

浅拷贝:

address是普通的引用类型,不拷贝,所以指向的是同一个Address

深拷贝:

因为浅拷贝是默认进行的，所以引用类型需要自己重写方法里面写一下。将其变成深拷贝。

例子：

@Override
public Object clone() throws CloneNotSupportedException{
	Person person = (Person)super.clone();
    Address cloneAddress = (Address)super.clone();
    person.setAddress(cloneAddress);
    return person;
}

Java的值拷贝(传递)

/*
"值拷贝":基本类型
"地址拷贝":引用类型
*/
public class TestValueCopy{
	
	public static void main(String[] args){
		
		int a = 10;
		int b = 20;
		
		Person p1 = new Person("zhangdan",true,24);
		Person p2 = new Person("lisi",false,33);
		
		//基本类型的值拷贝:将数值拷贝一份传递到方法中去操作
		swap(a,b);
		//引用类型的值拷贝:将引用变量拷贝一份传递到方法中操作
		swap(p1,p2);
		addAge(p1);
			
		System.out.println("a="+a);
		System.out.println("b="+b);
		System.out.println("p1 =>"+p1);
		System.out.println("p2 =>"+p2);
		
	}
	
	//只是将值传递过来,这个方法的操作与上面的a,b无关.
	public static void swap(int a,int b){
		int temp = a;
		a = b;
		b = temp;
	}
	//只是将引用变量传递过来,这个方法的操作与上面的p1,p2无关.
	public static void swap(Person p1,Person p2){
		Person temp = p1;
		p1 = p2;
		p2 = temp;
	}
	
	//只是将引用变量传递过来,但是两个引用变量指向的是同一个对象,所以值改变了
	public static void addAge(Person p){
		int age = p.getAge();
		age += 10;
		p.setAge(age);
	}

}
