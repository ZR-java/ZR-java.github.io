系统相关类

java.lang.System

System也是我们经常用的类，目前为止，我们用的最多的就是它下面out属性对象的println方法，项目其实用得最多的还是它的一些可以获取系统信息的一些方法。

import java.util.*;
import java.io.*;

public class TestSystem{
	
	public static void main(String[] args) throws Exception{
		
		Map<String,String> envMap = System.getenv();
        //static Map<String,String>	getenv() 返回当前系统环境的不可修改的字符串映射视图。
		Set<Map.Entry<String,String>> entrySet = envMap.entrySet();
		for(Map.Entry<String,String> element:entrySet){
			System.out.println("key="+element.getKey()+",value="+element.getValue());
		}
		
		System.out.println("==============");
		
		System.out.println(System.currentTimeMillis());//返回当前时间（以毫秒为单位）。
		
		System.out.println(System.getenv("JAVA_HOME"));
		
		//获取所有的系统属性
		Properties props = System.getProperties();
		//将所有系统属性保存到props.txt文件中
		props.store(new FileOutputStream(System.getProperty("user.home")+"/props.txt") , "System Properties");
		//props.store(new FileOutputStream("C:\\workspace\\01.JavaSE\\day09\\props.txt") , "System Properties");
		//输出特定的系统属性
		System.out.println(System.getProperty("os.name"));
	}
	
}

java.util.Scanner

通过运行java程序时，只能在程序运行之前预规定好参数， 稍微再复杂一点的程序，这种获取参数的方式就显得有点 太简单了，我们希望能够通过键盘输入的方式获取一些参 数信息，这里的Scanner类就能满足我们的要求

Scanner主要提供了两个方法来扫描输入

  • hasNextXxx()：

    判断是否还有下一个输入项，其中Xxx可以是Int、Long等代表的基本数据类型字符串，如果需要判断是否有下一个字符串，则可以直接使用hasNext()

  • nextXxx()：

    获取下一个输入的值。Xxx的意思跟上面的一 样。

import java.util.Scanner;

public class TestScanner{
	
	public static void main(String[] args){
		
		for(String arg:args){
			System.out.println(arg);
		}
	
        //构造从键盘获取值的Scanner对象
		Scanner scanner = new Scanner(System.in);
		scanner.useDelimiter("\r\n");
        //Scanner	useDelimiter(String pattern) 将此扫描器的分隔模式设置为从指定的构造的模式 String 。
		String oneline = scanner.next();
		System.out.println(oneline);
	
	}
	
}

注意：Scanner默认使用空白（空格、Tab制表、回车） 作为多个输入项之间的分隔符

使用Scanner之后,输入一串字符，却只拿到了hello,因为默认是以空格，回车，Tab键作为分隔符。所以一行被分为三个字符串。

我们可以使用回车\r和换行\n作为分隔符。

scanner.useDelimiter("\r\n");

以上是键盘扫描输入，Scanner还可以扫描文件来输入

  • Scanner类不仅仅只能读取单个字符、数字等，还可以读取一行

  • Scanner也提供了两个方法来逐行读取

    boolean hasNextLine()：输入源中是否还有下一行

    String nextList()：返回输入源中下一行的字符串

import java.util.Scanner;
import java.io.*;  //后面需要输入输出流,先*上

public class TestScanner{
	
	public static void main(String[] args) throws Exception{
		
		for(String arg:args){
			System.out.println(arg);
		}
		
		File file = new File("D:\\workspace\\JavaStudy\\JavaSE\\17.Java常用类\\1.txt"); //正斜杠/就不要转义了,\反斜杠就需要转义.
		Scanner scanner = new Scanner(file);
		while(scanner.hasNextLine()){
			System.out.println(scanner.nextLine());
		}
		
	}
	
}

字符串相关类

String、StringBuffer、StringBuilder

  • String：String是不可变类，即一旦一个String对象被创建后，包含在 这个对象中的字符序列是不允许改变的，一旦想改变，只有创建新的 对象，直到这个对象被JVM垃圾回收销毁
  • StringBuffer：StringBuffer对象代表一个可以改变的字符序列，当一 个StringBuffer对象被创建后，可以通过append、insert、reverse、 setCharAt、setLength等一些方法来改变这个字符串的值，直到最后 通过调用toString方法再变成String类型的值
  • StringBuilder：StringBuilder与StringBuffer之间很类似，唯一的不 同，就是StringBuilder是线程不安全的，而StringBuffer是线程安全 的，所以由此可以知道StringBuilder的效率还要比StringBuffer高
import java.util.Scanner;

public class TestString{
	
	public static void main(String[] args){
		
		String str = "name="+"张三"+",gender="+"男"+",age="+"24";
		/*
		一共11个对象
		中间过渡的字符串,浪费
		name=张三
		name=张三,gender=
		name=张三,gender=男
		name=张三,gender=男,age=
		*/
		
		StringBuffer sb = new StringBuffer();
		sb.append("name=")
		  .append("张三")
		  .append(",gender=")
		  .append("男")
		  .append(",age=")
		  .append("24");		//此时还不是字符串
		//用这个方法只有六个对象
		String str2 = sb.toString();
		
	}
	
}

Math类

java.lang.Math

Java已经提供了基本的+、-、 * 、/、%等一系列的数学操 作符，但是更为复杂的操作符，如三角函数、对数运算则无能为力了。这个时候的java.lang.Math类就有作用了

查看java.lang.Math的API文档

public class TestMath{
	
	public static void main(String[] args){
		
		System.out.println("Math.E = "+Math.E);
		//E 自然对数的底数e
		System.out.println("Math.PI = "+Math.PI);
		//圆周率
		System.out.println("Math.random = "+Math.random());
		//static double	random() 返回值为 double值为正号，大于等于 0.0 ，小于 1.0 。
		System.out.println("Math.round(3.45) = "+Math.round(3.45));
		System.out.println("Math.round(3.65) = "+Math.round(3.65));
		//四舍五入
		//static long	round(double a) 返回参数中最接近的 long ，其中 long四舍五入为正无穷大。
		System.out.println("Math.rint(3.65) = "+Math.rint(3.65));
		System.out.println("Math.rint(3.45) = "+Math.rint(3.45));
		//static double	rint(double a) 返回与参数最接近值的 double值，并且等于数学整数。
		System.out.println("(int)3.45 = "+(int)3.45);
		//直接强转,特点:精度丢失
		
	
	}
	
}

Random类

java.util.Random

  • Random类专门用于生成一个伪随机数
  • 构造器
    • 使用默认种子构造一个随机数类
    • 使用指定的long类型的值作为种子生成一个随机数类
  • 灵活度要比Math类的random方法高
import java.util.Random;

public class TestRandom{
	
	public static void main(String[] args){
		
		//该方法有参(种子),只有第一次调用才生成伪随机数
		//一个种子对应一套序列,种子一样,序列一样.
		Random random1 = new Random(1000L);
		for(int i=0;i<10;i++){
			System.out.println(random1.nextInt(100));
		}
		
		System.out.println("====================");
		
		//该无参方法,随着系统纳秒时间的变化而种子(seed)变化,
		//所以没调用一次生成的伪随机数都不一样.
		Random random2 = new Random();
		for(int i=0;i<10;i++){
			System.out.println(random2.nextInt(100));
		}
		
	}
	
}

推荐：

使用当前的系统时间来作为随机数的种子，或者使用默认的构造器。（无参构造器）

日期处理类

java.util.Date

  • Java提供了java.util.Date类来处理日期
  • 由于该类存在的时间比较长，从JDK1.0开始就存在了，好多构造器已经被标识为已过时，剩下了两个构造器
    • Date() ：构造一个代表当前时间的Date对象
    • Date(long)：根据指定的long值来生成一个Date对象。long值指 的是从公元1970.1.1 00:00:00时间点所经过的毫秒数

java.util.Date类的主要方法

  • boolean after(Date when)
    • 测试该日期是否在指定日期when之后
  • boolean before(Date when)
    • 测试该日期是否在指定日期when之前
  • int compareTo(Date anotherDate)
    • 比较两个日期的大小，该日期大于后面的就返回正数，相等返回0 ，小于就返回负数
  • boolean equals(Object obj)
    • 当前两个时间表示同一时刻时就放回true
  • long getTime()
    • 返回毫秒数
  • void setTime(long time)
    • 设置该Date对象的时间

Date - > Calendar 用void setTime(Date date)使用给定的 Date设置此日历的时间。

Calendar - > Date 用Date getTime() 返回一个表示此 Calendar 时间值（从历元至现在的毫秒偏移量）的 Date 对象。

java.util.Calendar

  • 因为Date类型很多的构造器与方法都不推荐再使用，所以在此不再推荐使用Date类型，而是使用Calendar类
  • Calendar是一个抽象类，表示特定环境下的日历
  • 获取Calendar实例的方法
    • Calendar cal = Calendar.getInstance();

java.util.Calendar类的常用方法

  • void add(int field,int amount)
    • 根据日历的规则，将指定的时间量添加或减去给定的日历字段。
  • void roll(int field,int amount)
    • 向指定日历字段添加指定（有符号的）时间量，不更改更大的字段。
  • int get(int field)
    • 返回给定日历字段的值。
  • int getActualMaximum(int field)
    • 给定此 Calendar的时间值，返回指定日历字段可能具有的 Calendar 。
  • int getActualMinimum(int field)
    • 给定此 Calendar的时间值，返回指定的日历字段可以具有的 Calendar 。
  • void set(int field,int value)
    • 将给定的日历字段设置为给定的值。
  • void set(int year,int month,int date)
    • 设置日历字段中的值 YEAR ， MONTH和 DAY_OF_MONTH 。
  • void set(int year,int month,int date,int hourOfDay,int minute,int second)
    • 设置字段中的值 YEAR ， MONTH ， DAY_OF_MONTH ， HOUR_OF_DAY ， MINUTE和 SECOND 。
import java.util.Calendar;
import java.util.Date;
import java.text.SimpleDateFormat;

public class TestCalendar{
	
	public static void main(String[] args){
		
		String pattern = "yyyy-MM-dd HH:mm:ss";
		SimpleDateFormat sdf = new SimpleDateFormat(pattern);
		
		Date date = new Date();
		//System.out.println(date);//不符合国人习惯
		System.out.println(sdf.format(date));
		
		Calendar cal = Calendar.getInstance();
		int year = cal.get(Calendar.YEAR);
		int month = cal.get(Calendar.MONTH);
		int day = cal.get(Calendar.DAY_OF_MONTH);
		int hour = cal.get(Calendar.HOUR_OF_DAY);
		int minute = cal.get(Calendar.MINUTE);
		int second = cal.get(Calendar.SECOND);
		
		//System.out.println(year+"-"+(month+1)+"-"+day+" "+hour+":"+minute+":"+second);
		//以上注释的都是废话,应该用SimpleDateFormat
		
	}
	
}

SimpleDateFormat对日期进行格式化输出

java.text.SimpleDateFormat

  • SimpleDateFormat是DateFormat的子类，可以更简单、 更直观地格式化和解析日期对象 SimpleDateFormat sdf = new SimpleDateFormat(“yyyy年MM月dd日 HH时 mm分ss秒”); System.out.println(sdf.format(new Date()));
import java.util.Calendar;
import java.util.Date;
import java.text.SimpleDateFormat;

public class TestCalendar2{
	
	public static void main(String[] args) throws Exception{
		
		String pattern = "yyyy-MM-dd HH:mm:ss";
		SimpleDateFormat sdf = new SimpleDateFormat(pattern);
		
		Date date = new Date(2008-1900,7,8,20,8,8);
		
		Calendar cal = Calendar.getInstance();
		cal.setTime(date);//Date->Calendar
		cal.add(Calendar.MONTH,1);//往后加一个月,会自动进位
		//cal.roll(Calendar.MONTH,1);//不进位
		
		//获得对应域的最大值
		System.out.println(cal.getActualMaximum(Calendar.DAY_OF_MONTH));
		
		System.out.println(sdf.format(cal.getTime()));
		//Calendar转成Date后用format格式化
		
		String dateString = "1999-01-09 12:12:12";//这个date字符串必须满足sdf定义的pattern
		Date date2 = sdf.parse(dateString);
		System.out.println(date2);
		
	}
	
}

一般情况下先使用Calendar处理日期，再转换成Date 类，再由SimpleDateFormat类格式化输出

Java程序的国际化（I18N）

国际化相关的Java类

java.util.ResourceBundle

  • 用于加载一个资源包

java.util.Locale

  • 对应一个特定的国家/区域、语言环境

java.text.MessageFormat

  • 用于将消息格式化

国际化资源文件

  • 为实现程序的国际化，必须提供程序所需要的资源文件。 资源文件的内容由key-value对组成
  • 资源文件的命名可以有3种格式
    • basename_language_country.properties
    • basename_language.properties
    • basename.properties

如:

msg_zh_CN.properties简体中文资源文件

msg_en_US.properties美式英文资源文件

msg_en.properties英文资源文件

msg.properties默认的资源文件

若资源文件包含非西方字符，则需要用JDK自带的工具来处理，native2ascii，这个工具的语法格式如下。

  • native2ascii 资源文件名 目标资源文件名
import java.util.*;
import java.text.MessageFormat;

public class TestI18N{
	
	public static void main(String[] args){
		
		//获得系统默认的国家/语言环境
		Locale myLocale = Locale.getDefault();
		//根据指定的国家/语言环境加载对应的资源环境
		ResourceBundle bundle = ResourceBundle.getBundle("msg",myLocale);
		
		//获得本地化字符串
		String name = bundle.getString("name");
		System.out.println(name);
		
		String msg = bundle.getString("msg");
		System.out.println(MessageFormat.format(msg,"zhurui"));
	}
	
}
