2021-10-28- 2022-03-09

JavaSE-Java

接口

  • 接口不是类，而是对类的一组需求的描述
  • 如果类遵从某个特定的接口，那么就履行这项服务
  • 合同、规定、协议
  • 接口没有实例
  • 接口可以有方法和域
    • 方法默认修饰符：public abstract
    • 域默认修饰符：public static final

java不支持多继承。

标记接口，例如Cloneable接口（里面没有方法）

抽象类可以不用实现接口里面的方法

接口的作用

  • 访问的统一
  • 程序的动态性
  • 实现C++多继承功能、为类提供新的类型
  • 为多态提供了实现的技术前提

接口的定义

关键字:implements

StudentDao.java

/*
Dao(DAO):Date Access Object
数据访问对象
*/
public interface StudentDao{
	
	/*public static final*/String VERSION = "1.0";
	
	/*public abstract*/Student find(int id);
	Student[] find(String name);
	
}

StudentDaoImpl.java (实现接口里面的所有方法)

public class StudentDaoImpl implements StudentDao{
	
	@Override
	public Student find(int id){
		if(id==1){
			return new Student("zhurui",true,24);
		}else if(id==2){
			return new Student("zhangsan",true,25);
		}else{
			return null;
		}
	}
	
	@Override
	public Student[] find(String name){
		Student[] array = new Student[10];
		if(name.contains("z")){
			array[0] = new Student("zhangsan",true,24);
			array[1] = new Student("zhangwuji",true,20);
			array[2] = new Student("zhangsanfeng",true,200);
		}else if(name.contains("l")){
			array[0] = new Student("lisi",true,24);
			array[1] = new Student("libai",true,45);
			array[2] = new Student("limochou",false,35);
		}
		return array;
	}
	
}

接口的特性

  • 接口不是类，不能使用new关键字实例化一个接口
  • 能声明接口的变量
  • 接口变量必须指向实现了接口的类的对象
  • 使用instanceof关键字判断一个对象是否实现了某个特定的接口
  • 接口可以被继承
  • 一个接口也可以继承多个父接口 (接口可以多继承.)
  • 接口不可以被final关键字修饰

再议instanceof

public class TestStudentDao2{
	
	public static void main(String[] args){
		
		StudentDao studentDao = new StudentDaoImpl();
		
		System.out.println(studentDao instanceof Object);
		//studentDao这个接口引用变量所指向的对象是 Object类型 的一个实例 true
		System.out.println(studentDao instanceof StudentDaoImpl);
		//......引用变量所指向的对象是 StudentDaoImpl类 的一个实例 true
		System.out.println(studentDao instanceof StudentDao);
		//.......引用变量所指向的对象是 StudentDao接口实现类 的一个实例 true
		
	}
	
}

接口与抽象类的区别⭐⭐⭐⭐⭐

抽象类:

  • 属于类的范畴，可以单独存在
  • 不可以实例化对象
  • 可以被子类继承（子类必须实现父抽象类中定义的抽象方法）
  • 默认的成员变量是普通成员变量
  • 其中可以存在抽象方法，也可以不存在抽象方法
  • 子类继承的父类只能有一个

接口：

  • 属于和类平级的概念，可以单独存在
  • 不可以实例化对象
  • 可以被抽象类、普通类实现
  • 默认的成员变量是静态常量(final)
  • 其中的方法全部都是抽象方法，全部都没有方法体
  • 子接口继承的父接口可以有多个

接口可以多实现和多继承
