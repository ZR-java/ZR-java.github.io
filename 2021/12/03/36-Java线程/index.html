    线程与进程

    进程

    • 操作系统能同时做很多事情，由不同的进程来完成
    • 人类的视觉反应时间是0.1秒，在一个时间点上，程序是单个运行的
    • 进程有自己独立的CPU时间片、代码和数据
    • 各个进程之间没有任何的影响

    线程

    • 线程是相对于进程而言的
    • 多线程是共同存在于一个进程中的
    • 线程有自己独立的CPU时间片，但是可以共享代码和数据
    • 进程依赖操作系统
    • 线程依赖进程

    多线程的概念

    在Java中，不可能用多进程来处理一些应用

    Java从语言级支持多线程

    在Java中，一切都是对象，所以线程也是一个对象，这个对象也有相对应的类

    在Java中，线程是由三部分组成的

    • cpu时间片
    • 数据
    • 线程的代码

    Java的多线程

    线程对象与线程的区别

    • 线程对象代表了一个线程，但并不是说一个线程对象就是一个线程
    • 我们不可能绕开JVM直接创建线程，我们只能访问JVM中的线程对象，线程是由JVM来启动的
    • 线程对象就是我们访问系统底层的线程的一个代理，当我们调用start()方法之后线程才会启动

    多线程编程的两种方式

    继承Thread类，覆盖run()方法

    • run()方法中写的代码就是这个线程的代码
    • 要启动线程,必须调用start()方法

    实现Runnable接口

    • 实现run()方法
    • new一个Runnable对象
    • 构造一个线程对象
    • Thread t = new Thread(Runnable对象);

    线程的三要素

    • CPU时间片 :
      • 继承Thread类即可
    • 线程代码 :
      • 覆盖Thread类的run方法,或者实现Runnable接口的run方法
    • 数据 :
      • 一般情况下,继承Thread父类的方式没有数据
      • 只有实现了Runnable接口的方式才会有数据

    继承Thread与实现Runnable接口两种编码的区别

    • 继承的好处：编码简单、可读性好
    • 实现接口的好处：保留了类的继承，代码和逻辑分离，便于实现多线程
    • 真正的多线程代码应该使用实现Runnable接口的方式

    线程的相关方法

    void setPriority(int n)

    • 设置优先级，可以是1~10，默认是5，数据越大，优先级 越高

    并不是每个操作系统都会按照优先级来选择线程，有些操作系统并不看这个优先级

    • 独占式操作系统：按优先级
    • 共享式操作系统：不按优先级来

    由于需要跨平台，所以最好不要通过优先级来决定线程 的执行顺序。

    在独占式系统中，高优先级进入可运行状态，会打断正在运行的低优先级的线程，高优先级线程将进入运行状态

    static void yield() 也叫让位方法

    • 暂停线程，优先级大于等于本类的线程才能去抢系统时间片，低于本类的没机会抢到，并不是所有的操作系统都支持线程的优先级，Windows7、Windows8.1、 Windows10、Linux不支持。

    this.yield()

    • 只会让位优先级高的，如果看不出来，只能是系统对线程的优先级支持的不是很完善

    void setDaemon(boolean on)

    • 将该线程标记为守护线程或用户线程，守护线程依赖于其他的线程，当虚拟机结束的时候，该线程也会结束

    static void sleep(long millis)

    • 暂停线程，本线程不会去抢系统时间片，等sleep完了才会去抢系统时间片

    void join()

    • 等待该线程终止，（即先让该线程运行，其他线程暂停）

    void join(long millis)

    • 等待该线程终止的时间最长为millis毫秒

    void join(long millis, int nanos)

    • 等待该线程终止的时间最长为millis毫秒+nanos纳秒

    让本线程加入到主线程中去,相当于单线程执行

    要先启动线程才能join，否则无效

    线程的同步

    不推荐,但是简单:

    //这个synchronized锁的对象一般是当前对象
    public synchronized void run1() {   //(同步方法) 此方法简单,但是效率低下,相当于一整个方法都串行
        try {
            //模拟网络延迟
            Thread.sleep(50);
            //获取账户中的余额
            int balance = this.balance;
            //余额不足
            if (balance<500){
                throw new RuntimeException("余额不足");
            }
            balance = balance - 500;
            //模拟网络延迟
            Thread.sleep(50);
            this.balance = balance;
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }

    推荐:

    @Override
    public void run() {
        //加个this表示锁的是当前对象
        synchronized (this) {   //(同步代码块)
            try {
                //模拟网络延迟
                Thread.sleep(50);
                //获取账户中的余额
                int balance = this.balance;
                //余额不足
                if (balance < 500) {
                    throw new RuntimeException("余额不足");
                }
                balance = balance - 500;
                //模拟网络延迟
                Thread.sleep(50);
                this.balance = balance;
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }

    • 线程的栈空间是独立的，堆空间是共享的，所以多个线程都可以访问堆空间中的对象

      • 栈空间里面放的是run()方法里的变量，堆空间里面放的是线程对象的成员变量。

    • 原子操作，几个不可分割的操作，要么一起做，要么不做

    • 多线程并发访问共享资源，如果不加同步控制，打断了原子操作，会造成共享资源的数据冲突

      • 数据不一致
      • 数据不完整

    synchronized

    • synchronized(对象 o）{代码块}
    • 只有获得o的锁标记的线程才能进入代码块
    • 当代码块执行结束后，o的锁标记就会被自动释放
    • 注意同步代码块锁的是对象，而不是代码块
    • 对o加锁的同步代码块，只有取得o对象的锁标记的同步 代码块才能进入
    • 代码块中一般会是对o进行的原子操作
    • 当一个线程失去CPU时间片，进入可运行状态，或者进入阻塞状 态时候，不会释放拥有的锁标记

    synchronized第二种用法

    在方法的声明中加入synchronized修饰符

    • public synchronized void method(){}
    • synchronized作为方法修饰符时，在整个方法范围内对this加锁， 哪个线程能获得this的锁标记，就能调用这个方法

    线程的同步的应用

    回忆区别

    • ArrayList、Vector
    • HashMap、HashTable
    • StringBuilder、StringBuffer

    List list = Collections.syncronizedList(new ArrayList());

    • 外同步，同步的方式在外部完成

    List list = new Vector();

    • 内同步，类本身方法同步

    线程在同步中的状态

    • 每一个对象不仅仅有个锁标记，他还有一个锁池
    • 锁池中存放着等待该对象锁标记的线程
    • 当一个线程希望获得对象的锁标记，却拿不到的时候，这个线程会进入该对象的锁池。我们称这个线程进入了锁池状态
    • 当对象的锁被释放之后，锁池中的线程有好多时，谁能得到对象的锁标记呢？（去抢）
    • 当锁池中的一个线程得到锁标记后，进入什么状态呢？（可运行状态）
    • 进入运行状态只有一条路，从可运行状态被系统选中
    • 谁能出锁池
      • 操作系统会从锁池中挑选一个线程进入可运行，操作系统有可能会选出等待时间较长线程
      • 对象本身没有选择的权利

    线程的同步注意事项

    • 同步代码的嵌套

      • CPU在某一时刻永远只执行一个线程

        • 一个对象的锁标记只能给一个线程，但是一个线程可以同时拥有多个对象的锁标记，即同步代码块可以嵌套。

          synchronized(o1){//拿到o1的锁标记
    synchronized(o2){//这个线程同时拥有o1和o2的对象的锁标记
        //...
    }
}//避免这样写

    • 当一个线程阻塞在锁池中时，不会释放拥有的其他锁标记(死锁)。

    方法java.lang.Object的

    wait()

    • 必须出现在同步代码中
    • 线程会释放所拥有的所有的锁标记，而且肯定至少有一个锁被释放
    • 线程退出运行状态，进入对象的等待队列（这是一个对象的第三个固有的组成部分）
    • 等待队列也是一个阻塞的状态

    notify()

    • 必须出现在同步代码中
    • 从当前的等待队列中释放一个线程（notifyAll为释放所有线程），一般会用notifyAll
    • 不会释放自己拥有的对象的标记锁

    notifyAll()

    • notifyAll()会把所有的线程都从等待队列中释放出来，出来后的线程进入什么状态？
    • 进入锁池状态
      • 无论状态如何转变，都不能改变程序执行的流程，所以从等待队列中释放出来的线程还是得得到锁标记才能执行wait后面的代码， 所以它只能回到锁池中等待对象的锁标记
    • 使用if还是使用while
      • 在多生产者或多消费者的情况下，要通过while反复判断，以确保生产在它生产时栈一定是不满的

    等待队列和锁池的区别

    • 线程要获得锁标记，而得不到锁标记的则进锁池，是迫不得已的
    • 等待队列中的线程是主动进入的
    • 锁池中的线程手里还可能有锁标记
    • 等待队列中的线程手里一定没有当前对象的锁标记
    • 等待队列中的线程如果没有调notify的话，永远出不去， 只有当调用了notify后，才会出去。
